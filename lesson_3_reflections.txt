When would you want to use a remote repository rather than keeping all your work local?

  Удаленный репозиторий удобно использовать когда над проектом работает несколько
  человек. Или можно использовать его как пример своей работы, доступный для
  просмотра всем желающим.

Why might you want to always pull changes manually rather than having Git automatically
stay up-to-date with your remote repository?

	В локальном репозитории могут быть файлы или отдельные изменения, которые не нужно
	выкладывать в публичный доступ, например какие-то личные наработки, либо незаконченная
	версия продукта. В таком случае разумно наличие только ручной синхронизации.
	Я неправильно понял вопрос: в случае ручного pull, это разумно по причине того
	что репозитории могут быть довольно значительными по размеру, с изменениями от других
	авторов, при этом тебе нужно иметь локальную версию только со своими изменениями
	или какую-то конкретную ветку.

Describe the differences between forks, clones, and branches.
When would you use one instead of another?

	Fork используется когда нужно создать копию имеющегося репозитория со ссылкой на оригинал,
	клон - просто скопировать имеющийся репозиторий с remote на локальную машину или в пределах
	локальной машины. Branch - создание ветки проекта для множеста разных целей, без потери
	возможности обращения к основной версии.

What is the benefit of having a copy of the last known state of the remote stored locally?

	При наличии копии можно работать без связи с сетью, например, в путешестии, с более-менее
	актуальной версией remote репозитория. К тому же можно сравнивать версии файлов в локальной
	ветке и удаленной.
